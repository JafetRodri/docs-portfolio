Executive Summary

This standard defines the required labeling baseline for network equipment deployed or managed by Meridian Professional Services. The goal is faster identification and troubleshooting through uniform label content, placement, and cable labeling. It applies to firewall, switch, and access point deployments, with conformance verified during provisioning and handoff.

1. Purpose and Scope

1.1 Purpose

Establish a single, testable standard for labeling network equipment across Meridian-delivered environments.

1.2 Scope

Applies to: Meridian-owned and client-managed network devices handled by Professional Services — firewalls, switches, and wireless access points, including device, port, and power-cable labels.

Does not apply to: non-network devices, or temporary lab labeling unless promoted to production.

5. Standard Requirements

Key Requirement

All in-scope equipment must follow the label media, placement, format, and evidence requirements below. Each requirement maps to a control framework.

5.1 Label Media and Placement

STD-5.1.1 Label Tape Size

  • Requirement: Use 1/2 inch label tape as the default; 3/4 inch only where 1/2 inch is unsuitable, using two-line formatting.
  • Evidence: Installation photos and implementation checklist.
  • Control Mapping: ISO 27001:2022 A.5.9; ITIL v4 Service Configuration Management.

STD-5.1.2 Alignment

  • Requirement: Labels applied level and aligned with a visible device edge where practical.
  • Control Mapping: NIST CSF 2.0 ID.AM-01.

5.2 Firewall Labeling

STD-5.2.1 Hostname Label

  • Requirement: Firewalls labeled front and back with hostname (e.g., FW-HQ-01).
  • Control Mapping: ISO 27001:2022 A.5.9.

STD-5.2.2 Port Labels

  • Requirement: LAN/WAN ports labeled using P<port#>: LAN <IPv4> and P<port#>: WAN <IPv4> formats. Failover labeled WAN2.
  • Control Mapping: ISO 27001:2022 A.8.9.

5.3 Switch and Access Point Labeling

STD-5.3.1 Switch Uplink

  • Requirement: Uplink port labeled Uplink: P<port#>; static management IP labeled Mgmt IP: <IPv4>.
  • Control Mapping: ITIL v4 Service Configuration Management.

STD-5.3.2 Access Point and PoE

  • Requirement: Access points labeled with hostname; any PoE injector labeled with the associated AP hostname.
  • Control Mapping: ISO 27001:2022 A.5.9.

6. Exceptions and Risk Acceptance

Exceptions may be approved only when physical constraints prevent compliant placement, the risk is documented and accepted by the designated authority, compensating controls are recorded, and a sunset date and remediation plan are defined. Exceptions are tracked to closure in the exception log.
