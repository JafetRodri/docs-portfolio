New Workstation Provisioning

This SOP describes the standard procedure for the Service Desk to provision and hand over a new Windows workstation to an end user, from imaging through final verification.

Doc ID: MMS-SD-SOP-021 Last updated: 2026-05-08 Est. time: 30–45 minutes TLP:GREEN — Limited disclosure, restricted to the community
Quick Answer

Image the device with the standard build, join it to the domain, confirm the RMM and EDR agents are active, install the standard application set, complete the new-user profile setup, then verify all checks and hand over to the user with the onboarding summary.

Before You Begin

  • An assigned asset tag and the user's account details.
  • Access to the standard imaging build and the application deployment catalog.
  • Network connectivity for domain join and agent enrollment.
Single-Owner Procedure

This procedure is performed end-to-end by one Service Desk technician. Because a single role owns every step, the steps are linear with no per-step handoff metadata — contrast this with a multi-team SOP, where each step carries its own owner.

Procedure

1
Image the workstation

Apply the current standard Windows build using the imaging tool. Confirm the build version matches the approved baseline before proceeding.

2
Join the domain

Join the workstation to the client domain and confirm it appears in the directory under the correct organizational unit.

3
Confirm management agents

Verify the RMM and EDR agents are installed, active, and reporting. The device should appear in both consoles before the build is considered managed.

Info

If either agent is missing, deploy it manually and re-check before continuing — an unmanaged endpoint must never be handed to a user.

4
Install the standard application set

Deploy the standard application catalog for the user's role. Confirm each application launches and is licensed.

5
Complete user profile setup

Configure the user's profile, email, and any role-specific access. Confirm first sign-in succeeds and the profile loads correctly.

6
Verify and hand over

Run the verification checklist below. Once all checks pass, hand the device to the user with the onboarding summary and record completion in the asset record.

Tip

Walk the user through first sign-in in person where possible — it resolves most day-one tickets before they're raised.

Verification Checklist

CheckExpected Result
Domain joinDevice visible in directory, correct OU
RMM agentActive and reporting in console
EDR agentActive, protected, policy synced
ApplicationsStandard set installed and licensed
User sign-inProfile loads, email connected
